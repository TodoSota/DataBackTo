# 実装内容
実装の際、まず"最低限の完成"でよいことを意識する。
つまり過度に複雑なアルゴリズムでなく、最低限移動や自分以外のオブジェクトに対して影響を及ぼすことができればそれだけで構わない。
またUIや登場キャラクターについても素材などは考えずにひとまずプリミティブにて対応する。
とりあえずゲームループの分だけ完成すればそこから適宜追加する。

## ステージ選択
テンプレートのステージ選択画面なので、担当者の裁量に任せます。

## ステージ内
- 基本的にその存在の持っている内部的な数値(ステータス)及びそれらの増減(ダメージ処理など)は Status に
- 基本的なふるまい(移動や攻撃などのアクション)は Controller へとクラス分離して実装する

なおアイテムは取得された時の効果くらいしか持ちえないのでわざわざ分離しなくてもいい場合はその限りでない

- 後に拡張や大きく手直しする可能性のある場所は <最低限> と文頭につける。
  - なお手直しと書くとネガティブな感じがするが、ひとまず関数を用意してこの段階でそのデータ構造と関数群でうまくいくかを検証しているので、あくまで関数や変数を確定させた部分に意義があり、根本的な手直しはしないで済むというところまでは持っていきたい。
- 今やると面倒なのであとで考えましょうかの部分には <追加> とつける。

### Enemy
ライフ制(のように数値を扱う)。基本的にザコ敵は "1ライフ" のみ。後述するが接触しても、殴られても一発で死ぬ。
攻撃については種類それぞれで違うので、それによる Player_HP の減少量や攻撃範囲、持続時間などは個別に対応すること。

- <最低限>移動アルゴリズム左右に移動する。自分の立っている足場の端で折り返すでも、一定距離右に行けば、一定距離左に。でも構わない
- <最低限>攻撃行動。もう一種敵を用意してこの部分だけ追加した敵を仮置きしてもよし。攻撃方法はなんでもいいが、Playerに影響を与えられることを確認する
- 接触ダメージ。Playerに接触した際に互いにダメージを負うようにする。Player側にノックバックなどがないのでハメ防止のために敵は1-3発あたりで撤収する
- <追加>ドロップ。倒されたときに何かしらドロップしてもいい。一定確率やレアドロップとなると面倒なので追加とする

### Item

[コイン]
Player が貯めていくアイテム。プレイヤーのアクションの際に消費され、レシートに記録され、と割と扱いの難しい存在。
実装に際してはルール通りに値の変更や追加をやっていくだけ。

- 取得。ステージ上に存在し、プレイヤーが接触した場合プレイヤーのMoneyの値を増加させる。
- <追加>コインの種類。1,10,100円などの価値違いについてはまた後程。多分そこまで追加もだるくはない

### Player
HP は 100 。この最大値については増減はない。
 → 歩行やジャンプなどのアクションのたびに徐々に減少していく。
 → 敵からの攻撃被弾時に減少する。減少量は敵の攻撃力次第。
コイン という収集量のステータスを持つ。
 → 各種アクションの際に減少する。
レシート発行 というアクションがあり、"発動時の自分の状況"を記録する
レシート消費 というアクションにて、"記録時の自分の状況"に自分のステータスを書き換える。
 → 保存上限は 3 
 → 保存内容は HP , ジャンプ回数 , 所持金額 , 状態異常(後述)

- <最低限>移動。左右移動のことWASDにて、といっても左右移動のみなので気にしなくていい。とりあえず移動速度についてはとりあえず固定値でいい。状態のときに拡張しよう
- <最低限>ジャンプ。Spaceキーにて発動。上に飛ぶ。2段ジャンプが上限。
- <最低限>自傷ダメージ。移動やジャンプの着地の際に徐々にHPが減少する。減少量についてはひとまず固定値で。
- <最低限>通常攻撃。目の前の相手に攻撃する。XCVBNMキーのようなSPACEの上あたりのキーすべて。どれかを押せばいい。持続時間や攻撃範囲などは固定値でいいが、変数は置いてほしい
- <最低限>レシート発行。状態を記録する。正直なんで最低限にしてるかはわからないが、多分あとから調整したくなると思う。
- <最低限>レシート使用。状態を再現する。発行と使用が同じボタンでのアクションなので、それぞれの発動時間云々の調整があると思う。
- <最低限>状態変化。ステータス(アクションに関わるステータス含む)の変化が起きる。帯電状態のみ。それがなにかについてはWordファイル参照
- <最低限>ヒップドロップ。ジャンプ中に攻撃ボタン押下で発動。着地時に自傷ダメージ。攻撃範囲に若干の横幅を持たせて、強化状態に応じてその横幅を変えたいので、攻撃数値と横幅(縮尺)については変数を置いてほしい。
- 強化通常攻撃。Shiftを押しながら攻撃押下でコイン発射。コインを飛ばす。ダメージこそ1なものの、だいたいの敵はライフ1なので牽制用といいつつ即死攻撃をばらまいている。飛距離については一定距離で消えるでも、画面端で消去でもなんでもいい。とりあえず直線上に飛ばしてほしい。お金がない場合は発動しない。
- 強化ジャンプ。Shiftを押しながらSpace押下で発動。"現在のジャンプ回数分" のお金が消費される。ジャンプの上限回数を超えてジャンプできる。お金が尽きた場合はもう無理ってなる。
- <最低限>強化ヒップドロップ。shift押しつつ、ジャンプ中に攻撃ボタン押下で発動。着地ダメ無効、ダメージアップ。なお必要金額未満の所持金であれば発動しない。必要金額については未定なのでそれらしい数値を配置する。
